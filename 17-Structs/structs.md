## 结构体是什么
结构体（struct）：把多个相关字段打包成一个整体的数据类型，便于传递、维护和扩展
- 结构体的类型名称
- 字段列表

## 创建结构体和使用
```go
type Student struct {
    ____ ____
    ____ ____
}

s := Student{name: "Tom", age: 18}

// 年龄
var age = s.age
s.age = 20
```

## 结构体指针
在函数中，直接传结构体参数时，传的是副本。所以在函数内部，修改结构体的字段，不会影响到调用者。
为什么需要结构体指针?
- 传结构体值：会复制一份结构体数据
- 传结构体指针：传的是地址，通常不复制整份结构体内容

```go
s := Student{name: "Tom", age: 18}
p := &s
// *p 才是那个实际的 Student
// 获取年龄
(*p).age
// 在书写层面上，允许使用 p.age 来访问结构体字段
// 这是因为 Go 会自动解引用指针，等价于 (*p).age
```

## 继承
Go 没有像 Java / C++ 那样的“类继承”
Go 更常用的是：
- 组合（composition）
- 嵌入（embedding）

传统 OOP 里的继承，通常强调的是：
- 复用已有字段/行为
- 扩展新的字段/行为
- 有时还包含一种 “是一个（is-a）” 的关系

```go
//包含：就是把另一个结构体当普通字段放进去
type Student struct {
    person Person
    school string
}
// 假设Person里面有name字段，那么访问name字段
s.person.name (显示写法)
```
```go
// 嵌入：是一种更特殊的写法，它会带来更方便的字段访问方式，直接写类型
type Student struct {
    Person
    school string
}
// 假设Person里面有name字段，那么访问name字段
s.name  // 直接访问，字段提升
s.person.name (显式写法)
```

**如果 Student 自己也定义了一个 name 字段，同时又嵌入了 Person，而 Person 里也有 name，那你觉得这时候直接写：**
```go
s.name
```
- 没有冲突时：可以字段提升，直接 s.name
- 直接写 s.name 时，会优先拿到 Student 自己的 name
- 有冲突时：必须用 s.Person.name 来访问

## 方法和嵌入的关系
Go 的方法 = 函数 + 接收者
在 Go 里，方法不是写在结构体定义内部，而是写在结构体定义外部，但是要指定接收者（receiver）

值接收者 Student
- 更适合只读场景
- 语义上更像“拿一份值来用”

指针接收者 *Student
- 更适合需要修改原对象
- 也适合避免复制大结构体

**如果一个方法的接收者是 *Student，而你手里有的是一个普通变量 s（不是指针），你觉得调用时更可能是哪种体验？**
- (&s).方法()，这个思路在语义上是通的
- 如果 s 是一个可取地址的普通变量，Go 通常允许你直接写：s.方法()
