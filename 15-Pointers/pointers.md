# Go 语言指针 (Pointers)

指针是一个变量，它存储了另一个变量的**内存地址**。

## 核心操作符
1. `&` (取地址)：获取一个变量的内存地址。
2. `*` (解引用/取值)：获取指针所指向地址中存储的具体数值。

## 为什么要用指针？
- **修改原始数据**：Go 是值传递（Pass by Value），如果想在函数内修改外部变量，需要传递指针。
- **性能优化**：对于大型结构体，传递指针比拷贝整个数据结构更高效。

## 其他说明
变量：像一个盒子，里面装着值
地址：这个盒子放在内存里的位置
指针：不是盒子本身，而是“记着盒子地址的一张纸条”

普通变量保存“值”
指针变量保存“地址”

## 问题

**如果原来 x = 10，p 指向 x，然后你把 *p 改成 20，你觉得 x 会不会变？为什么？**
在当前这个时刻，x 的值是 10 p 指向 x 所以读取 *p 时，得到的是 10

**为什么不用 *p 来表示，而是用 x？**
x：我直接知道这个变量本人是谁
*p：我先拿到地址，再顺着地址找到它
所以如果你已经直接有 x，当然写 x 更自然、更简单。这涉及到间接访问和直接访问的问题，但他们都访问的是同一块数据

**你觉得 int 和 *int 的区别是什么**？
int：变量里直接放 整数值
*int：表示“指向 int 的指针类型”
- int → 盒子里装数字，
- *int → 盒子里装“某个数字盒子的位置”

**如果 p 的类型是 *int，那 *p 的类型你觉得会是什么？为什么？**
对 p 解引用后，拿到的是它指向的那个值 既然它指向的是 int，那么 *p 对应的就应该是 int 这一层

**如果有一个 int 类型变量 x，那表达式 &x 你觉得得到的是什么类型？为什么？**
- &x 是一个表达式
- p 是一个变量

如果 p 保存了 &x，那才有：
- p 的类型是 *int
- p 的内容是 x 的地址

## 注意事项
- **空指针**：指针的零值是 `nil`。对 `nil` 指针进行解引用会触发 runtime panic。
- **不支持算术运算**：Go 的指针不像 C 语言那样可以进行 `p++` 操作（除非使用 `unsafe` 包）。
